# 🚀 Инструкции по загрузке на GitHub

Этот документ содержит пошаговые инструкции для инициализации Git репозитория и загрузки проекта на GitHub.

## Шаг 1: Инициализировать локальный Git репозиторий

```bash
cd /path/to/user_wallets

# Инициализировать Git
git init

# Добавить все файлы в staging area
git add .

# Сделать первый коммит
git commit -m "Initial commit: User Wallets REST API with concurrent transaction support"
```

## Шаг 2: Создать репозиторий на GitHub

1. Откройте https://github.com/new
2. Введите название: `user-wallets-api`
3. Добавьте описание: "REST API for managing user wallets with concurrent transaction support"
4. Выберите Public (для видимости при собеседовании)
5. **НЕ** инициализируйте с README (у нас уже есть)
6. Нажмите "Create repository"

## Шаг 3: Добавить remote и загрузить код

Скопируйте команду из GitHub (должна выглядеть как-то так):

```bash
# Добавить remote (замените USERNAME на ваш GitHub username)
git remote add origin https://github.com/USERNAME/user-wallets-api.git

# Переименовать ветку на main (если используется master)
git branch -M main

# Загрузить код на GitHub
git push -u origin main
```

## Шаг 4: Проверить результат

Откройте https://github.com/USERNAME/user-wallets-api в браузере и убедитесь, что:
- ✅ Все файлы загружены
- ✅ README.md отображается корректно
- ✅ Лог коммитов видна
- ✅ GitHub Actions workflow запущена (вкладка "Actions")

Когда вы показываете проект:
